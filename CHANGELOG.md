# Version Format

NexOS follows Semantic Versioning.

```text
MAJOR.MINOR.PATCH

Example:

1.0.0
1.1.0
1.1.1
2.0.0
```

- **MAJOR** → Breaking changes
- **MINOR** → New features
- **PATCH** → Bug fixes
